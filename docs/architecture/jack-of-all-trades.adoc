== Graph-Theoretic Specification

*Implementation-Agnostic Design*  

This document defines the complete system as a **directed, functional graph** that is independent of any specific programming language or runtime. It is engineered to be directly realizable in **Jaseci/Jac** (nodes → `node`, edges → `++>` / `-->` / `?:`, walkers → pure functional traversals, `by llm()` → typed decision vertices).  

The design prioritizes referential transparency, explicit data flows, and native graph operations, making it ideal for agentic reasoning over portfolio state while satisfying the Agent Client Protocol (ACP) and Model Context Protocol (MCP).

=== System Architecture

The system follows a three-tier architecture:

1.  **Front End ("JOAT"berg terminal)**: Replacing the role of DataSpell for user input and visualization. Acts as the ACP client.
2.  **JOAT (Jac of all Trades) ACP agent**: The central orchestrator, implementing the ACP server and MCP client logic.
3.  **Alpaca MCP server**: The tool provider for trading and market data, communicating with the JOAT ACP agent via **Interprocess Communication (IPC)**.

=== Graph Definition

[stem]
++++
G = (V, E, L)
++++

* *V*: Finite set of nodes (entities with state and attributes).  
* *E*: Finite set of directed edges (explicit, typed relationships).  
* *L*: Labeling function that assigns attributes, types, and metadata to nodes and edges.

The graph is **cyclic at the system level** (for continuous agentic loops) but **directed-acyclic within individual walker traversals** (enforced via visited flags and functional updates). All operations are **pure functions** over graph sub-structures: walkers consume a source node/sub-graph, compute, and produce a new immutable result that is attached via explicit edges.

=== Edges (E) — Explicit Directed Connections

All edges are **typed** and carry semantic labels. Walkers traverse them functionally (no mutation of source; new edges are created for updated state).

[cols="2,2,2,1,3", options="header"]
|===
| Source Node | Target Node | Edge Type / Label | Direction | Purpose / Semantics

| JOATberg terminal 
| JOAT ACP agent 
| `ACP_REQUEST` 
| → 
| JSON-RPC initialize / session/new / prompt

| JOAT ACP agent 
| JOATberg terminal 
| `ACP_RESPONSE` 
| ← 
| Serialized results + visualization payload

| JOAT ACP agent 
| AlpacaMCP-Server 
| `MCP_IPC_CALL` 
| ↔ 
| Interprocess communication (get_portfolio, place_order)

| JOAT ACP agent 
| PortfolioGraph 
| `HAS_PORTFOLIO` 
| ++> 
| Root ownership of entire portfolio sub-graph

| PortfolioGraph 
| Asset 
| `CONTAINS_ASSET` 
| ++> 
| Portfolio → individual securities

| Asset 
| Position 
| `HAS_POSITION` 
| ++> 
| Asset → user's current holding

| Position 
| TradeEvent 
| `HAS_TRADE_HISTORY` 
| --> 
| Position → chronological events

| TradeEvent 
| Asset 
| `AFFECTS` 
| --> 
| Trade impacts asset state

| PortfolioGraph 
| RiskNode 
| `HAS_RISK_PROFILE` 
| ++> 
| Global risk metrics attached to portfolio

| DecisionVertex 
| Asset / Position 
| `RECOMMENDS` / `EXECUTES` 
| --> 
| `by llm()` reasoning → concrete action

| JOAT ACP agent 
| VisualizationSink 
| `GENERATES_VIZ` 
| --> 
| Final output for JOATberg terminal
|===

*Functional Invariant*: Every edge creation is a pure operation returning a new edge object; the original graph remains immutable until a walker explicitly commits an update via a new edge.

=== Labels & Metadata (L)

* *Node Labels*: Each node carries typed attributes (e.g., `symbol: str`, `quantity: float`, `risk_score: float`).
* *Edge Labels*: Semantic verbs + constraints (e.g., `MCP_TOOL_CALL` includes `tool_name`, `parameters`, `permission_required`).
* *Walker Metadata*: Each walker declares input/output types and guarantees purity (no side effects except controlled external protocol calls).
* *Session Labels*: Transient labels on `JOAT ACP agent` for ACP session state (e.g., `session_id`, `mcp_config_passed_by_terminal`).

=== Phase 3: Slim New Layered InsightEngine + Exact Vibe Pipeline

The system now implements a 3-step functional pipeline for processing all user prompts:

1. **`get_portfolio_context(session_id)`**: Returns a `PortfolioContext` (frozen dataclass).
2. **`create_financial_insight(prompt, context)`**: Returns a `FinancialInsight` (graph structure).
3. **`create_visualization_sink(prompt, insight)`**: Returns the final `VisualizationSink`.

This pipeline is designed for direct migration to Jac walkers in Phase 4.
